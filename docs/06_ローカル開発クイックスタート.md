# ローカル開発クイックスタート

このドキュメントでは、FPV Drone Managerをローカルで開発するための手順を説明します。

## クイックスタート（5分）

### 1. 依存関係のインストール

```bash
cd /Users/atusi/repos/drone_spec
npm install
```

### 2. 環境変数の設定

```bash
# .env.exampleをコピー
cp .env.example .env.local

# .env.localを編集してFirebase設定を入力
```

`.env.local` の内容：

```bash
VITE_FIREBASE_API_KEY=your_api_key
VITE_FIREBASE_AUTH_DOMAIN=your_project_id.firebaseapp.com
VITE_FIREBASE_PROJECT_ID=your_project_id
VITE_FIREBASE_STORAGE_BUCKET=your_project_id.appspot.com
VITE_FIREBASE_MESSAGING_SENDER_ID=your_sender_id
VITE_FIREBASE_APP_ID=your_app_id
```

### 3. 開発サーバーの起動

```bash
npm run dev
```

ブラウザで http://localhost:3000 を開きます。

## 利用可能なコマンド

| コマンド | 説明 |

## Firebase Emulator（オプション）

本番Firebaseを使わずにローカルで開発する場合：

### 1. Firebase CLIのインストール

```bash
npm install -g firebase-tools
firebase login
```

### 2. Emulatorの起動

```bash
firebase emulators:start
```

Emulator UI: http://localhost:4000

### 3. アプリをEmulatorに接続

`src/lib/firebase/config.ts` に以下を追加（開発時のみ）：

```typescript
import { connectAuthEmulator } from 'firebase/auth'
import { connectFirestoreEmulator } from 'firebase/firestore'
import { connectStorageEmulator } from 'firebase/storage'

if (import.meta.env.DEV) {
  connectAuthEmulator(auth, 'http://localhost:9099')
  connectFirestoreEmulator(db, 'localhost', 8080)
  connectStorageEmulator(storage, 'localhost', 9199)
}
```

## 開発フロー

### 新しいページを追加する

1. `src/pages/` にページコンポーネントを作成
2. `src/App.tsx` にルートを追加
3. 必要に応じてナビゲーションを更新

### 新しいコンポーネントを追加する

1. `src/components/` に適切なフォルダを選択
2. コンポーネントファイルを作成
3. `index.ts` でエクスポート

### 新しいデータ型を追加する

1. `src/types/` に型定義を追加
2. `src/services/` にサービスを作成
3. `src/hooks/` にカスタムフックを作成


## トラブルシューティング

### 「Module not found: @/」エラー

**対処法**: パスエイリアスの設定を確認

`tsconfig.json`:
```json
{
  "compilerOptions": {
    "paths": {
      "@/*": ["src/*"]
    }
  }
}
```

### HMR（Hot Module Replacement）が動作しない

**対処法**:
1. 開発サーバーを再起動
2. ブラウザをハードリロード（Ctrl+Shift+R）

### Firebase関連のエラー

1. `.env.local` の設定を確認
2. Firebase Consoleで認証プロバイダが有効か確認
3. Firestoreデータベースが作成されているか確認
