# Firebase Authentication 設定ガイド

このドキュメントでは、FPV Drone ManagerアプリのためのFirebase Authenticationの設定手順を説明します。

## 前提条件

- [01_Firebaseプロジェクト作成ガイド](./01_Firebaseプロジェクト作成ガイド.md) の手順が完了していること

## Google認証の設定

本アプリではGoogleアカウントでのログインを使用します。

### 1. Authentication機能の有効化

1. [Firebase Console](https://console.firebase.google.com/) でプロジェクトを開く
2. 左メニューから「Authentication」を選択
3. 「始める」ボタンをクリック

### 2. Googleプロバイダの有効化

1. 「Sign-in method」タブを選択
2. 「Google」をクリック
3. 右上のトグルを「有効」にする
4. プロジェクトのサポートメールを選択（自分のGmailアドレス）
5. 「保存」をクリック

### 3. 承認済みドメインの確認

「Settings」タブ → 「承認済みドメイン」で以下を確認：

| ドメイン | 用途 |
|---------|------|
| `localhost` | ローカル開発 |
| `your-project-id.firebaseapp.com` | Firebase Hosting |
| `your-project-id.web.app` | Firebase Hosting（代替） |

カスタムドメインを使用する場合は、ここに追加します。

## 管理者アカウントの設定（オプション）

管理者用のメール/パスワード認証を設定する場合：

### 1. メール/パスワードプロバイダの有効化

1. 「Sign-in method」タブを選択
2. 「メール/パスワード」をクリック
3. 「メール/パスワード」を有効にする
4. 「保存」をクリック

### 2. 管理者ユーザーの作成

1. 「Users」タブを選択
2. 「ユーザーを追加」をクリック
3. 管理者用のメールアドレスとパスワードを入力
4. 「ユーザーを追加」をクリック

### 3. 管理者権限の付与

管理者権限はFirestoreのユーザードキュメントで設定します。
ユーザーが初回ログイン後、Firebase Consoleで直接ドキュメントを編集：

1. Firestore Database → `users` コレクション
2. 該当ユーザーのドキュメントを開く
3. `role` フィールドを `"admin"` に変更

## 認証フローの確認

### 正常な認証フロー

```
1. ユーザーが「Googleでログイン」をクリック
2. Googleの認証ポップアップが表示
3. ユーザーがGoogleアカウントを選択・認証
4. アプリにリダイレクト
5. usersコレクションにユーザードキュメントが作成/更新
6. ダッシュボードにリダイレクト
```

### テスト方法

1. 開発サーバーを起動（`npm run dev`）
2. http://localhost:3000 にアクセス
3. 「ログイン」ボタンをクリック
4. Googleアカウントで認証
5. ダッシュボードが表示されることを確認

## セキュリティ設定

### 推奨設定

| 設定項目 | 推奨値 | 説明 |
|---------|--------|------|
| メール列挙保護 | 有効 | ユーザーの存在を隠す |
| ユーザーアカウントのリンク | 有効 | 同一メールのアカウントを統合 |

### メール列挙保護の有効化

1. 「Settings」タブ → 「ユーザーアクション」
2. 「メール列挙保護」を有効化

## トラブルシューティング

### 「popup closed by user」エラー

**原因**: ユーザーがポップアップを閉じた、またはポップアップブロッカー

**対処法**:
- ブラウザのポップアップブロッカーを無効化
- `localhost:3000` を許可リストに追加

### 「auth/unauthorized-domain」エラー

**原因**: 現在のドメインが承認されていない

**対処法**:
1. Firebase Console → Authentication → Settings
2. 「承認済みドメイン」に現在のドメインを追加

### 「auth/network-request-failed」エラー

**原因**: ネットワーク接続の問題

**対処法**:
- インターネット接続を確認
- ファイアウォール設定を確認
- VPN使用時は一時的に無効化
