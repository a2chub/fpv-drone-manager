# デプロイガイド

このドキュメントでは、FPV Drone ManagerアプリをFirebase Hostingにデプロイする手順を説明します。

## 前提条件

- すべてのFirebase設定が完了していること
- Node.js 18以上がインストールされていること
- Firebase CLIがインストールされていること

## Firebase CLIのセットアップ

### 1. Firebase CLIのインストール

```bash
npm install -g firebase-tools
```

### 2. Firebaseにログイン

```bash
firebase login
```

ブラウザが開き、Googleアカウントでの認証を求められます。

### 3. プロジェクトの確認

```bash
# 現在のプロジェクトを確認
firebase projects:list

# プロジェクトを選択
firebase use your-project-id
```

## 初回デプロイの手順

### 1. Firebaseプロジェクトの初期化

```bash
cd /Users/atusi/repos/drone_spec

# Firebase初期化（初回のみ）
firebase init
```

選択項目：
- **Features**: Firestore, Hosting, Storage を選択
- **Firestore Rules**: `firestore.rules` を使用
- **Firestore Indexes**: `firestore.indexes.json` を使用
- **Public directory**: `dist` を入力
- **Single-page app**: `Yes` を選択
- **GitHub Actions**: お好みで（CI/CD用）

### 2. 本番用ビルド

```bash
npm run build
```

ビルド結果は `dist/` フォルダに出力されます。

### 3. デプロイの実行

```bash
# 全サービスをデプロイ
firebase deploy

# または個別にデプロイ
firebase deploy --only hosting          # Webアプリのみ
firebase deploy --only firestore:rules  # Firestoreルールのみ
firebase deploy --only storage:rules    # Storageルールのみ
```

### 4. デプロイ完了

デプロイが成功すると、以下のURLでアクセスできます：

- `https://your-project-id.web.app`
- `https://your-project-id.firebaseapp.com`

## 環境別のデプロイ

### 開発環境と本番環境の分離

異なるFirebaseプロジェクトを使用することを推奨：

```bash
# .firebaserc に複数プロジェクトを設定
{
  "projects": {
    "default": "fpv-drone-manager-dev",
    "production": "fpv-drone-manager-prod"
  }
}
```

```bash
# 開発環境へデプロイ
firebase use default
firebase deploy

# 本番環境へデプロイ
firebase use production
firebase deploy
```

## CI/CD（GitHub Actions）

### 自動デプロイの設定

`.github/workflows/firebase-hosting-merge.yml`:

```yaml
name: Deploy to Firebase Hosting on merge
on:
  push:
    branches:
      - main

jobs:
  build_and_deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'npm'

      - name: Install dependencies
        run: npm ci

      - name: Build
        run: npm run build
        env:
          VITE_FIREBASE_API_KEY: ${{ secrets.VITE_FIREBASE_API_KEY }}
          VITE_FIREBASE_AUTH_DOMAIN: ${{ secrets.VITE_FIREBASE_AUTH_DOMAIN }}
          VITE_FIREBASE_PROJECT_ID: ${{ secrets.VITE_FIREBASE_PROJECT_ID }}
          VITE_FIREBASE_STORAGE_BUCKET: ${{ secrets.VITE_FIREBASE_STORAGE_BUCKET }}
          VITE_FIREBASE_MESSAGING_SENDER_ID: ${{ secrets.VITE_FIREBASE_MESSAGING_SENDER_ID }}
          VITE_FIREBASE_APP_ID: ${{ secrets.VITE_FIREBASE_APP_ID }}

      - name: Deploy to Firebase Hosting
        uses: FirebaseExtended/action-hosting-deploy@v0
        with:
          repoToken: ${{ secrets.GITHUB_TOKEN }}
          firebaseServiceAccount: ${{ secrets.FIREBASE_SERVICE_ACCOUNT }}
          channelId: live
          projectId: your-project-id
```

### GitHub Secretsの設定

1. GitHub リポジトリ → Settings → Secrets and variables → Actions
2. 以下のシークレットを追加：

| Secret名 | 値 |
|---------|---|
| `VITE_FIREBASE_API_KEY` | Firebase API Key |
| `VITE_FIREBASE_AUTH_DOMAIN` | Firebase Auth Domain |
| `VITE_FIREBASE_PROJECT_ID` | Firebase Project ID |
| `VITE_FIREBASE_STORAGE_BUCKET` | Firebase Storage Bucket |
| `VITE_FIREBASE_MESSAGING_SENDER_ID` | Messaging Sender ID |
| `VITE_FIREBASE_APP_ID` | Firebase App ID |
| `FIREBASE_SERVICE_ACCOUNT` | サービスアカウントJSON |

### サービスアカウントの取得

1. Firebase Console → プロジェクト設定 → サービスアカウント
2. 「新しい秘密鍵の生成」をクリック
3. JSONファイルをダウンロード
4. ファイル内容を `FIREBASE_SERVICE_ACCOUNT` シークレットに設定

## プレビューデプロイ

プルリクエスト用のプレビュー環境：

```bash
# プレビューチャンネルにデプロイ
firebase hosting:channel:deploy preview-branch-name

# 有効期限付きでデプロイ（7日間）
firebase hosting:channel:deploy preview --expires 7d
```

## ロールバック

問題が発生した場合のロールバック：

### Firebase Console から

1. Firebase Console → Hosting
2. 「リリース履歴」タブ
3. ロールバックしたいバージョンの「...」→「ロールバック」

### CLI から

```bash
# リリース履歴を確認
firebase hosting:releases:list

# 特定のバージョンにロールバック
firebase hosting:rollback
```

## カスタムドメインの設定

### 1. ドメインの追加

1. Firebase Console → Hosting
2. 「カスタムドメインを追加」をクリック
3. ドメイン名を入力（例: `fpv-drone.example.com`）

### 2. DNS設定

表示されるDNSレコードをドメインレジストラで設定：

| タイプ | ホスト | 値 |
|-------|-------|---|
| A | @ | Firebase提供のIPアドレス |
| TXT | @ | 所有権確認用の値 |

### 3. SSL証明書

Firebase Hostingは自動的にSSL証明書を発行・管理します（無料）。

## パフォーマンス最適化

### ビルド最適化

`vite.config.ts`:

```typescript
export default defineConfig({
  build: {
    rollupOptions: {
      output: {
        manualChunks: {
          vendor: ['react', 'react-dom', 'react-router-dom'],
          firebase: ['firebase/app', 'firebase/auth', 'firebase/firestore'],
        },
      },
    },
  },
})
```

### キャッシュ設定

`firebase.json`:

```json
{
  "hosting": {
    "headers": [
      {
        "source": "**/*.@(js|css)",
        "headers": [
          {
            "key": "Cache-Control",
            "value": "public, max-age=31536000, immutable"
          }
        ]
      },
      {
        "source": "**/*.@(jpg|jpeg|gif|png|svg|webp)",
        "headers": [
          {
            "key": "Cache-Control",
            "value": "public, max-age=604800"
          }
        ]
      }
    ]
  }
}
```

## トラブルシューティング

### 「Not Found」エラー（直接URL アクセス時）

**原因**: SPAのルーティングが機能していない

**対処法**: `firebase.json` で rewrites を確認

```json
{
  "hosting": {
    "rewrites": [
      {
        "source": "**",
        "destination": "/index.html"
      }
    ]
  }
}
```

### デプロイが遅い

**対処法**:
1. `dist/` フォルダのサイズを確認
2. 不要なファイルを `.firebaseignore` に追加
3. 画像を最適化

### 「Permission denied」エラー

**原因**: Firebaseプロジェクトへのアクセス権がない

**対処法**:
1. `firebase login` を再実行
2. プロジェクトの権限を確認
